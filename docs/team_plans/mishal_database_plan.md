# Implementation Plan: Mishal (Firebase Database)

**Role:** Database Master
**Tech Stack:** Google Cloud Firebase/Firestore

## Overview
You are the master of the data. You take the messy JSON data created by Anum and Eman and organize it in Firebase so Anamta's AI can search it quickly.

## Step-by-Step Execution

### 1. Setup Firebase
- Create a new project in Google Firebase and enable "Firestore Database".
- Setup the main collections: `Clients`, `Providers`, and `Bookings`.

### 2. Organize the JSON Data
- Take the temporary JSON files generated by Anum (Client data) and Eman (Provider data).
- Upload this messy data into Firebase in an organized way. 

### 3. Critical Provider Attributes (Mandatory for AI)
*For Anamta's AI to work, the `Providers` collection MUST have these specific fields:*
- `location` (Lat/Lng for distance calculation)
- `rating` (0-5 stars)
- `cancellation_rate` (0.0 to 1.0)
- `on_time_score` (0.0 to 1.0)
- `base_hourly_rate` (PKR)

### 4. Write Database Queries
- Write the Firestore queries needed to find the best match when a user requests something.
- Example: *"Find all active Providers where Service = AC Repair".*
- You will share these queries and the Firebase connection info with Faiq and Anamta so their Python code can talk to your database.
